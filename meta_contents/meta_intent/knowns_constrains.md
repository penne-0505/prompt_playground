- instructionの数量制限は、通常使用において超過しない程度の余裕がある
- 一つのinstructionあたり、およそ(400 token / 1500文字)程度の制限が存在する。どちらかに引っかかった場合、エラーとなる
- instructionは、Gemini本体もしくは専用の低位なモデルによって解析が行われ、指示が曖昧だとそのモデルによって解釈され、より明確に書き換えられる
- 明示的な改行は無視される。同様に、箇条書きも口語的なものに変換されるか、改行部分のみがカットされる
- "Geminiへのカスタム指示"という一つのinstructionしか利用できず、instructions / memory などで分割することはできない
- おそらく、instruction内での変数展開はサポートされていない
